История версий FazTestBot
===========================

v1.0 (Исходная версия)
- Базовая реализация шифрования: Морзе, числовой шифр, QR-код
- Командный интерфейс через /start, /code
- Reply-клавиатуры для навигации

v1.1
- Добавлен шифр Цезаря с выбором языка и шага
- Улучшена обработка ошибок при вводе шага
- Добавлена поддержка буквы "ё" в словарях

v2.0 (Переход на Inline-меню)
- Полный переход с reply-клавиатур на inline-клавиатуры
- Единое сообщение-меню вместо спама сообщениями
- Добавлены разделы "Помощь" и "О боте"
- Реализована навигация "Назад" через стек состояний

v2.1
- Исправлена ошибка "сессия устарела" при повторном шифровании
- Реализовано удаление сообщений пользователя после ввода
- Улучшена структура состояний (хранение параметров шифрования)
- Добавлена кнопка "Изменить шаг" для шифра Цезаря

v2.2
- Исправлена работа с QR-кодом через класс QRCode
- Улучшена читаемость кода (упорядочены словари шифров)
- Добавлены комментарии с информацией о версии в начале файла
- Исправлено форматирование вывода результата Цезаря

v2.3
- Убрана ненужная информация в разделе "Помощь"
- Реализовано редактирование существующего сообщения вместо отправки нового
- Исправлен формат вывода Цезаря (результат сразу после заголовка)
- Заменена кнопка "Изменить шаг" на "Изменить настройки"

v2.4
- Возвращена простая генерация QR-кода через qrcode.make()
- Добавлено удаление сообщения пользователя при вводе шага Цезаря
- Упорядочены словари шифров с визуальным разделением на группы
- Улучшена обработка ошибок при генерации QR-кода
- Полная очистка чата от сообщений пользователя (единый поток диалога)

v2.5
- Полностью переработана логика работы с QR-кодом:
  * Бот удаляет своё сообщение с запросом ввода
  * Отправляет QR-код как отдельное сообщение-изображение
  * Отправляет отдельное сообщение "QR сгенерирован" с кнопками
  * При нажатии ЛЮБОЙ кнопки удаляется сообщение с QR-кодом
- Исправлена критическая ошибка в шифре Цезаря:
  * Бот больше не пытается редактировать удалённое сообщение пользователя
  * Все операции выполняются с сообщением бота (сохранённым в состоянии)
- Добавлена проверка алфавита в шифре Цезаря с понятными сообщениями об ошибках
- Универсальная очистка QR-кода при любом действии пользователя

v2.6
- Разделение кода на отдельные файлы:
  * Вынос конфигурации в config.py
  * Вынос истории версий в history.txt
- Оптимизация структуры кода:
  * Удаление дублирующейся информации из основного файла
  * Централизация констант и настроек
- Улучшение поддержки и читаемости кода
- Подготовка архитектуры для дальнейшей модульной оптимизации (версия 3.0)

v3.0 — Глобальное обновление архитектуры
- Полный переход на модульную архитектуру
- Создание бота-агрегатора для объединения функциональных модулей
- Новое главное меню: Модули, Помощь, О боте
- Изоляция модулей друг от друга (каждый модуль — независимый пакет)
- Единая точка входа (bot.py) для управления всеми модулями
- Первый модуль: "Шифратор" (адаптированная версия 2.6)
- Поддержка динамической регистрации новых модулей
- Оптимизированная структура файлов для масштабируемости

v3.1 — Логирование пользователей
- Добавлена база данных SQLite для хранения информации о пользователях
- Логируемые поля:
  * user_id — уникальный идентификатор пользователя
  * username — юзернейм (если указан)
  * full_name — полное имя (объединение first_name и last_name)
  * created_at — дата первого взаимодействия с ботом
  * last_active — дата последней активности
  * module_stats — статистика использования модулей (JSON)
- Автоматическое создание записи при первом взаимодействии
- Обновление last_active при каждом действии пользователя
- Инкремент счётчика использования модуля при шифровании
- Оптимизированная архитектура интеграции логирования в модули
- Полностью персональное использование — без политик приватности и согласий

v3.2 — Расширенное логирование операций шифрования
- Добавлена новая таблица cipher_operations для хранения полных операций шифрования
- Сохраняется ПОЛНЫЙ контент операций:
  * Исходный текст (до шифрования)
  * Результат шифрования (после обработки)
  * Тип шифра (Морзе, числовой, Цезарь, QR)
  * Параметры шифра (шаг и язык для Цезаря)
  * Дата и время операции с точностью до миллисекунд
  * Уникальный идентификатор операции (UUID)
- Автоочистка логов старше 30 дней (настраивается в config.py)
- Команда /my_stats для просмотра персональной статистики:
  * Общее количество операций
  * Распределение по типам шифров
  * Последние 5 операций с полным контентом
  * Статистика по датам (операций за сегодня/неделю/месяц)
- Логирование происходит ТОЛЬКО после успешного шифрования (ошибки не сохраняются)
- Полная история запросов доступна для персонального использования
- Оптимизированная структура БД для быстрого поиска и анализа

v3.4.1 — Скрытая админ-панель с визуальным интерфейсом
- Полностью скрытая система администрирования, недоступная обычным пользователям
- Два метода аутентификации:
  * Постоянный доступ по user_id (123456789)
  * Временный доступ (24 часа) через секретное кодовое слово "fazbear_admin_2026"
- Визуальное админ-меню с интуитивной навигацией через инлайн-кнопки
- Функции админ-панели:
  * 📊 Общая статистика по всем пользователям
  * 👤 Детальный профиль любого пользователя
  * 🔍 Поиск по логам шифрования по ключевым словам
  * 💾 Экспорт данных в CSV/JSON
  * 🧹 Ручная очистка логов (с подтверждением)
  * ⚙️ Управление настройками бота
  * 🚨 Экстренные действия (полная очистка)
- Скрытие админ-панели от обычных пользователей:
  * При попытке доступа к админ-функциям — бот игнорирует или отвечает "Не понимаю"
  * Нет никаких намёков на существование скрытых функций
- Обработка неизвестных команд для обычных пользователей
- Команда /my_stats перенесена в админ-панель (доступна только администраторам)
- Полная изоляция админ-логики от пользовательского интерфейса

v3.4.2 — Исправленная админ-панель с надёжной аутентификацией
- 🔧 ИСПРАВЛЕНО: Баг с поиском профиля по ID (блокировка бота после отмены поиска)
- 🔧 ИСПРАВЛЕНО: Временные сессии администратора теперь работают корректно
- 🔧 ИСПРАВЛЕНА: Опечатка "24 часа" → "24" в настройках сессии
- 🆕 НОВОЕ: Аутентификация отделена от интерфейса админ-панели:
  * /auth кодовое_слово — получение статуса администратора
  * /admin — вход в админ-панель (только для аутентифицированных)
- 🆕 НОВОЕ: Все пользователи (включая постоянных админов) попадают в обычное меню при /start
- 🆕 НОВОЕ: Состояния пользователей для режимов ввода (поиск, экспорт) с корректной отменой
- 🛡️ УЛУЧШЕНО: Безопасность — все админ-действия логируются, временные сессии автоматически истекают
- 🐞 УЛУЧШЕНО: Стабильность — устранены все критические баги предыдущей версии

v3.4.3 — Пасхалки и блокировка пользователей
- 🥚 НОВАЯ ФУНКЦИЯ: Пасхалка-головоломка с последовательной активацией:
  * Шаг 1: Отправить сообщение "Bot"
  * Шаг 2: Отправить сообщение "history"
  * Шаг 3: Отправить сообщение "open"
  * Награда: Открывается скрытое меню с историей разработки
- 🔒 Строгая проверка последовательности:
  * Сообщения должны быть отправлены в ТОЧНОМ порядке (1→2→3)
  * Любое другое сообщение сбрасывает прогресс
  * Пример: порядок 2→1→3 НЕ работает
- ⏱️ Cooldown 24 часа:
  * После успешной активации нельзя активировать снова в течение 24 часов
  * Автоматический сброс через сутки
  * Кнопка "🔙 Закрыть" в меню также сбрасывает состояние
- 📜 Скрытое меню разработки:
  * Кнопка "📜 История разработки" — показывает содержимое файла history.txt
  * Кнопка "🔙 Закрыть" — закрывает меню и возвращает в главное меню
  * Чтение истории напрямую из файла в директории бота
- 🚫 Система блокировки пользователей:
  * Список BLOCKED_USERS в config.py для указания заблокированных user_id
  * Полное игнорирование ВСЕХ сообщений от заблокированных пользователей
  * Бот не отвечает, не логирует, не создаёт записи в БД
  * Никаких уведомлений о блокировке (полная невидимость)
  * Расширяемая структура для будущего добавления причин блокировки

v4.0 (Текущая версия) — Модуль "URL Shortener"
- 🆕 НОВОЕ: Модуль сокращения ссылок по аналогии с шифрованием:
  * Бот запрашивает URL у пользователя
  * Пользователь отправляет ссылку
  * Бот удаляет сообщение пользователя
  * Бот редактирует своё сообщение с результатом
- 🆕 НОВОЕ: Поддержка двух API сервисов:
  * TinyURL (бесплатно, без API ключа)
  * Cutt.ly (бесплатно, требует API ключ, статистика кликов)
- 🆕 НОВОЕ: Формат вывода результата:
  * Исходная ссылка (с проверкой на длину)
  * Сокращённая ссылка (кликабельная)
  * Кнопки: "Открыть", "Другая ссылка", "Назад"
- 🆕 НОВОЕ: Валидация URL перед сокращением:
  * Проверка формата (http/https)
  * Проверка на допустимые символы
  * Авто-добавление https:// если отсутствует
- 🆕 НОВОЕ: История сокращённых ссылок в БД:
  * original_url — исходная ссылка
  * shortened_url — сокращённая ссылка
  * created_at — дата создания
  * clicks_count — количество кликов (если API поддерживает)
- 🆕 НОВОЕ: Меню "Мои ссылки":
  * Просмотр истории сокращённых ссылок
  * Отображение последних 10 ссылок
  * Отключение превью ссылок (disable_web_page_preview)
  * Специальная клавиатура с 2 кнопками
- 🛡️ ИСПРАВЛЕНО: Навигация в модуле:
  * Все кнопки "Назад" называются одинаково
  * Разные callback_data для разных действий
  * Корректное отображение списка модулей при возврате
  * Убрано дублирование кнопок "К списку модулей"
- 🛡️ ИСПРАВЛЕНО: Клавиатура в меню "Мои ссылки":
  * Только 2 кнопки: "🔗 Сократить ссылку" и "🔙 Назад"
  * Возврат в меню модуля, а не к списку модулей
- 🛡️ ИСПРАВЛЕНО: Отключение превью ссылок:
  * disable_web_page_preview=True во всех сообщениях
  * Предотвращение создания превью YouTube и других сайтов
  * Использование <code> для ссылок в списке
- 📊 УЛУЧШЕНО: Интеграция с экосистемой:
  * Наследование от BaseModule (v3.5 стандарт)
  * Автодискавери при загрузке
  * Параметр show_back_button для гибкости клавиатур
